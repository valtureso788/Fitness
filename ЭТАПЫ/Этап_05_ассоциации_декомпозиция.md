# Этап 5. Выделение ассоциаций. Декомпозиция предметной области

**Тема проекта:** Сервис фитнес-клуба (Абонементы, тренировки и посещаемость)  
**Дата выполнения:** 24.04.2026  

---

## 1. Перечень ассоциаций

| № | Ассоциация | Описание |
|:--|:---|:---|
| 1 | Клиент владеет Абонементом | Клиент приобретает абонемент для доступа к занятиям |
| 2 | Клиент оформляет Запись | Клиент бронирует место на тренировке |
| 3 | Тренер проводит Тренировку | Тренер назначается ответственным за занятие |
| 4 | Тренер отмечает Запись | Тренер фиксирует присутствие клиента |
| 5 | Администратор формирует Тренировку | Администратор создаёт расписание |
| 6 | Администратор управляет Абонементом | Администратор выдаёт и замораживает абонементы |
| 7 | Тренировка содержит Записи | Тренировка агрегирует все бронирования |
| 8 | Запись списывает с Абонемента | При посещении уменьшается остаток занятий |

## 3. Схема ассоциаций

```mermaid
classDiagram
    class Client {
        Клиент
    }
    class Coach {
        Тренер
    }
    class Admin {
        Администратор
    }
    class Subscription {
        Абонемент
    }
    class Training {
        Тренировка
    }
    class Booking {
        Запись
    }

    Client "1" --> "0..*" Subscription : владеет
    Client "1" --> "0..*" Booking : оформляет
    Coach "1" --> "0..*" Training : проводит
    Coach "1" --> "0..*" Booking : отмечает явку
    Admin "1" --> "0..*" Training : формирует
    Admin "1" --> "0..*" Subscription : управляет
    Training "1" *-- "0..*" Booking : содержит
    Booking "0..*" --> "1" Subscription : списывает
```

## 5. Описание подсистем

| Подсистема | Сущности | Назначение |
|:---|:---|:---|
| **Модуль пользователей** | User, Client, Coach, Admin | Доступ к системе, роли |
| **Модуль расписания** | Training | Сетка расписания, вместимость |
| **Модуль абонементов** | Subscription | Учёт прав на посещение |
| **Модуль записей** | Booking | Связующий модуль: бронирование и посещаемость |

---

## 6. Диаграмма последовательности записи на тренировку

```mermaid
sequenceDiagram
    participant К as Клиент
    participant МП as Модуль пользователей
    participant МР as Модуль расписания
    participant МА as Модуль абонементов
    participant МЗ as Модуль записей
    К->>МП: Авторизация
    МП-->>К: Доступ подтверждён
    К->>МР: Запрос расписания
    МР-->>К: Список тренировок
    К->>МЗ: Запрос на запись
    МЗ->>МР: Проверка мест
    МР-->>МЗ: Места есть
    МЗ->>МА: Проверка абонемента
    МА-->>МЗ: Абонемент валиден
    МЗ-->>К: Запись подтверждена
```
